📋 Prasyarat

Sebelum melakukan proses deployment, pastikan Anda telah menyiapkan komponen berikut:

  • Akun GitHub dan kode proyek ini sudah diunggah (push) ke repositori Anda.
  • Akun Vercel yang terhubung dengan akun GitHub Anda.
  • Database PostgreSQL Cloud. Karena Vercel adalah Serverless (yang bersifat ephemeral/sementara), Anda tidak bisa menggunakan database lokal (localhost). Anda wajib menggunakan database PostgreSQL cloud. Karena Anda menggunakan Supabase, ikuti petunjuk berikut:
    • Gunakan Connection Pooling (Transaction Mode): Di dashboard Supabase Anda, masuk ke Settings -> Database -> Connection string. Salin URL dengan opsi Transaction (biasanya menggunakan port 6543). Gunakan URL ini sebagai DATABASE_URL Anda di Vercel.
    • Menggunakan koneksi langsung (port 5432) pada arsitektur serverless Vercel berisiko cepat menghabiskan limit koneksi database Supabase Anda.

🛠️ Langkah 1: Memahami Konfigurasi yang Telah Disiapkan

Proyek backend ini telah dikonfigurasi agar siap dideploy langsung ke Vercel dengan penambahan dan modifikasi berkas berikut:

1. Berkas Konfigurasi Rute Vercel

NEW vercel.json

Mengatur agar semua request HTTP diarahkan ke serverless function entrypoint kita.

{
  "version": 2,
  "rewrites": [
    {
      "source": "/api/(.*)",
      "destination": "/api/index.ts"
    },
    {
      "source": "/(.*)",
      "destination": "/api/index.ts"
    }
  ]
}

2. Berkas Entrypoint Serverless

NEW api/index.ts

Berkas ini bertindak sebagai jembatan yang mengimpor aplikasi Express dan mengekspornya agar Vercel dapat mengubahnya menjadi Serverless Function.

import app from "../src/main/app.js";

export default app;

3. Script Otomasi Prisma

MODIFY package.json

Menambahkan script "postinstall": "prisma generate" agar Vercel secara otomatis men-generate Prisma Client yang cocok dengan lingkungan server Vercel pada saat instalasi dependencies.

"scripts": {
  "dev": "tsx src/main/server.ts",
  "build": "tsc",
  "start": "node dist/main/server.js",
  "test": "vitest run",
  "postinstall": "prisma generate"
}

4. Penyesuaian TypeScript

MODIFY tsconfig.json

Mengecualikan folder api dari kompilasi lokal tsc agar tidak memicu error kompilasi karena file api/index.ts berada di luar direktori src.

"exclude": ["prisma/seed.ts", "vitest.config.ts", "api"]

💾 Langkah 2: Menyiapkan Database PostgreSQL Cloud

Berikut adalah langkah-langkah membuat database di Neon.tech (gratis & mudah):

  1. Buka Neon.tech dan daftarkan diri menggunakan akun GitHub Anda.
  2. Buat proyek baru, beri nama (misal: spp-database), pilih versi PostgreSQL terbaru, dan tentukan regional server terdekat (misal: Singapore / Asia).
  3. Setelah database dibuat, Anda akan mendapatkan Connection String / URI. Salin string koneksi tersebut yang berformat seperti ini: 
    postgresql://username:password@ep-cool-shadow-123456.us-east-2.aws.neon.tech/neondb?sslmode=require
  4. Simpan link koneksi ini dengan aman untuk digunakan pada langkah selanjutnya.

📤 Langkah 3: Push Perubahan ke GitHub

Kirimkan perubahan konfigurasi di atas ke repositori GitHub Anda:

# Buka Terminal/PowerShell di folder spp-backend
git add .
git commit -m "chore: configure project for vercel deployment"
git push origin main
💡 Info: Pastikan seluruh perubahan file konfigurasi seperti vercel.json, api/index.ts, package.json, dan tsconfig.json sudah ikut ter-upload.

🚀 Langkah 4: Hubungkan ke Vercel

  1. Masuk ke dashboard Vercel.
  2. Klik tombol Add New... lalu pilih Project.
  3. Cari dan pilih repositori GitHub spp-backend Anda, kemudian klik Import.
  4. Pada halaman Configure Project:
    • Framework Preset: Pilih Other.
    • Root Directory: Jika Anda menggunakan subfolder di git, sesuaikan foldernya (misal: arahkan ke spp-backend). Jika struktur project ada di root terluar git, biarkan default (./).
  5. Buka menu dropdown Environment Variables dan tambahkan variabel-variabel berikut:
    Nama Variabel Nilai (Value) Keterangan
    DATABASE_URL [URI Database Cloud Anda dari Langkah 2] Koneksi ke PostgreSQL cloud Anda.
    JWT_SECRET [Tulis string random yang aman & panjang] Kunci rahasia enkripsi JWT Token login.
    NODE_ENV production Menandakan mode production.
  6. Klik tombol Deploy.
  7. Tunggu beberapa menit hingga proses instalasi dependency dan deploy selesai. Vercel akan menghasilkan URL publik Anda (misal: https://spp-backend-example.vercel.app).

🗄️ Langkah 5: Menjalankan Migrasi Database

Setelah Vercel sukses melakukan deploy, database cloud Anda di Neon/Supabase masih kosong dan tidak memiliki tabel. Anda perlu mengunggah skema tabel Anda ke database cloud tersebut.

⚠️ Penting: Jangan jalankan perintah ini jika belum memasukkan DATABASE_URL cloud ke dalam environment lokal Anda.

Cara Migrasi dari Komputer Lokal (Sangat Direkomendasikan)

  1. Di komputer lokal Anda, buka file .env di dalam direktori spp-backend (buat filenya jika belum ada).
  2. Ubah nilai DATABASE_URL dengan link database cloud Anda: 
    DATABASE_URL="postgresql://username:password@ep-cool-shadow-123456.us-east-2.aws.neon.tech/neondb?sslmode=require"
  3. Buka terminal lokal Anda pada direktori spp-backend, kemudian jalankan perintah untuk migrasi tabel Prisma: 
    npx prisma migrate deploy
    Perintah ini akan membaca folder prisma/migrations dan membuat seluruh struktur tabel di database cloud.
  4. Untuk mengisi data awal database (seperti admin default, dsb.), jalankan perintah seeding: 
    npx prisma db seed

🧪 Langkah 6: Uji Coba Deployment

Buka browser, Postman, atau aplikasi testing API lainnya untuk menguji endpoint Anda di Vercel.

Akses salah satu rute API Anda menggunakan URL Vercel yang didapatkan, contohnya:

GET https://nama-proyek-anda.vercel.app/api/categories

Jika rute tersebut terproteksi, Anda akan mendapatkan respons status 401 Unauthorized (artinya routing Express berfungsi). Jika rute bersifat publik, Anda akan mendapatkan data JSON. Pastikan Anda tidak mendapatkan status 404 Not Found atau 500 Internal Server Error.

⚠️ Catatan Penting untuk Serverless di Vercel

  • Cold Starts: Serverless function akan "tidur" jika tidak ada request masuk selama beberapa waktu. Request pertama setelah masa pasif mungkin memakan waktu 1-2 detik untuk merespons karena proses inisialisasi ulang.
  • Serverless Timeout: Di akun Vercel gratis (Hobby), limit waktu maksimal pemrosesan satu request API adalah 10-15 detik. Pastikan query data Anda tidak memakan waktu terlalu lama.
  • Connection Pooling: Sistem serverless dapat meluncurkan puluhan container baru secara bersamaan jika traffic melonjak tinggi. Ini dapat membanjiri database PostgreSQL Anda (membuat batas maksimal koneksi terlampaui). Disarankan menggunakan connection pooler seperti PgBouncer atau mengaktifkan fitur pooling di penyedia cloud database Anda (misalnya menggunakan port 6543 di Supabase).
