📋 Prasyarat

Sebelum melakukan proses deployment ke Render, pastikan Anda telah menyiapkan komponen berikut:

  • Akun GitHub dan kode proyek ini sudah diunggah (push) ke repositori Anda.
  • Akun Render yang terhubung dengan akun GitHub Anda.
  • Database Supabase. Karena Anda sudah memilki database di Supabase, Anda hanya perlu menyalin *Connection String* database tersebut.
💡 Info: Berbeda dengan Vercel yang bersifat serverless (cepat mati & hidup), Render menjalankan aplikasi Anda sebagai server tradisional yang aktif terus menerus (Web Service). Konfigurasi serverless seperti vercel.json dan api/index.ts tidak digunakan pada platform ini.

💾 Langkah 1: Mendapatkan Connection String Supabase

Karena Render merupakan server persisten, Anda dapat menggunakan URL koneksi standar maupun pooling dari Supabase:

  1. Masuk ke dashboard Supabase dan buka proyek database Anda.
  2. Masuk ke menu Project Settings (ikon roda gigi di pojok kiri bawah) kemudian pilih tab Database.
  3. Gulir ke bawah ke bagian Connection string dan pilih tab URI.
  4. Salin string koneksi tersebut.
    • Rekomendasi untuk Render: Anda bisa menggunakan opsi Session (port koneksi langsung 5432) atau opsi Transaction (port pooling 6543). Keduanya dapat bekerja dengan stabil di Render.
  5. Ganti bagian [YOUR-PASSWORD] pada string koneksi tersebut dengan password database Supabase Anda yang sebenarnya.

🛠️ Langkah 2: Mengkonfigurasi Build & Start Command

Render memerlukan perintah spesifik untuk mempersiapkan (*build*) berkas JavaScript Anda dan perintah untuk menjalankannya (*start*):

1. Build Command

Mempersiapkan dependensi, membuat Prisma Client, dan mengompilasi TypeScript menjadi JavaScript biasa:

npm install --include=dev && npx prisma generate && npm run build

2. Start Command

Menjalankan aplikasi hasil kompilasi yang terletak di dalam direktori dist:

npm run start

(Skrip ini memanggil node dist/main/server.js secara internal sesuai isi package.json Anda)

🚀 Langkah 3: Membuat Web Service di Render

  1. Masuk ke dashboard Render.
  2. Klik tombol New + di sudut kanan atas halaman lalu pilih Web Service.
  3. Pilih opsi Build and deploy from a Git repository lalu klik Next.
  4. Hubungkan dengan akun GitHub Anda, cari repositori spp-backend, lalu klik Connect.
  5. Pada halaman pengaturan Web Service:
    • Name: Masukkan nama layanan Anda (misal: spp-backend).
    • Region: Pilih regional terdekat dengan pengguna Anda (sangat disarankan memilih Singapore).
    • Branch: Pilih branch utama Anda (misal: main).
    • Runtime: Pilih Node.
    • Build Command: npm install --include=dev && npx prisma generate && npm run build
    • Start Command: npm run start
    • Instance Type: Pilih paket Free (atau paket berbayar sesuai kebutuhan produksi).
  6. Klik tombol Advanced lalu pilih Add Environment Variable untuk memasukkan variabel lingkungan berikut:
    Nama Variabel Nilai (Value) Keterangan
    DATABASE_URL [URI Database Supabase Anda dari Langkah 1] Koneksi string database Supabase Anda.
    JWT_SECRET [Tulis string random yang aman & panjang] Kunci enkripsi untuk JWT Token autentikasi.
    NODE_ENV production Status lingkungan produksi aplikasi.
  7. Klik Create Web Service. Proses deploy akan segera dimulai secara otomatis.

🗄️ Langkah 4: Menjalankan Migrasi Database

Agar database Supabase Anda memiliki seluruh struktur tabel yang didefinisikan dalam kode backend, Anda perlu menjalankan migrasi database.

Metode A: Migrasi Otomatis Saat Build di Render (Sangat Direkomendasikan)

Mengintegrasikan migrasi database ke dalam alur build Render. Jika migrasi gagal, build akan gagal dan mencegah naiknya kode yang rusak ke produksi.

Ubah kolom Build Command di pengaturan Render Anda menjadi:

npm install --include=dev && npx prisma generate && npx prisma migrate deploy && npm run build

Metode B: Migrasi Manual dari Komputer Lokal

  1. Di komputer lokal Anda, buka file .env di dalam direktori spp-backend (buat berkas tersebut jika belum ada).
  2. Ubah nilai DATABASE_URL dengan link koneksi Supabase Anda: 
    DATABASE_URL="postgresql://postgres:password@db.supabase.co:5432/postgres"
  3. Buka terminal lokal Anda pada direktori spp-backend, kemudian jalankan perintah migrasi: 
    npx prisma migrate deploy
  4. *(Opsional)* Jalankan skrip seed jika ingin menyisipkan data awal: 
    npx prisma db seed

🧪 Langkah 5: Uji Coba Deployment

Setelah status build di Render berubah menjadi Live (berwarna hijau), Render akan memberikan URL publik untuk aplikasi Anda (misal: https://nama-aplikasi.onrender.com).

Uji endpoint backend Anda menggunakan Postman atau browser dengan mengakses:

GET https://nama-aplikasi.onrender.com/api/categories

Jika berhasil terhubung, Anda akan menerima respon data JSON atau status error autentikasi 401 Unauthorized dari Express (bukan error server 502 Bad Gateway).

⚠️ Catatan Penting untuk Layanan Free Tier Render

  • Spin Down (Tidur Otomatis): Pada paket Free, jika aplikasi Anda tidak menerima request apapun selama **15 menit**, Render akan menonaktifkan server sementara. Saat ada request masuk berikutnya, server memerlukan proses inisialisasi ulang yang memakan waktu **30 - 50 detik** untuk aktif kembali (Cold Start).
  • Batas Jam Aktif Bulanan: Akun gratis memiliki batasan durasi jam aktif bulanan yang dibagikan ke seluruh Web Service Anda. Jika limit habis, Web Service Anda akan ditangguhkan hingga bulan berikutnya.
  • Untuk menjaga aplikasi selalu aktif terus-menerus dan responsif secara komersial, disarankan melakukan upgrade ke paket berbayar terendah (**Starter Plan** seharga sekitar $7/bulan).
